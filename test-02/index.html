And we... write Powerpoint

Blaming PowerPoint for bad presentations is about as fair as blaming fountain pens for bad poetry.

The real problem with PowerPoint and its open source equivalents...

...is that they aren't web-friendly.

When you export a slideshow to present on the web...

...what you actually get is a bunch of images.

Your original text is lost: all you have are pixels arranged in the shapes of letters.

If you want something people can replay on the web, you have to make a video...

...which is probably hundreds of times larger than your original slides...

...and doesn't contain the actual text of your presentation either.

Browsercast is our solution to this problem.

It lets you replay an HTML5 slideshow with a voiceover.

The audio playback controls and the "next slide/previous slide" controls are synchronized...

...and if you pause this presentation, you'll see that you can select the text, or use "View Source" to see what we're saying.

What's more, because it's HTML5, it can easily be restyled using CSS to play on mobile devices...

...and since it's just text and audio, it's a fraction the size of a conventional screencast.

Browsercast is open source, so if you'd like to help us move it forward, please fork our GitHub repo or mail us.
